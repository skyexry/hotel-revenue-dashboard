# Hotel Revenue Intelligence Platform

> An end-to-end BI platform that ingests hotel booking data, computes revenue KPIs, runs a dynamic pricing engine, and surfaces insights through an interactive dark-theme dashboard.

---

## Background

Hotel revenue managers spend hours each week pulling data from disparate systems, computing occupancy and ADR by hand, and assembling pricing recommendations in spreadsheets. This platform automates that workflow: raw booking records flow through a cleaning and standardization pipeline, feed into a KPI engine that calculates RevPAR, ADR, and cancellation rates by month and channel, then pass through a rule-based pricing engine that flags months requiring rate adjustments. A single-page dashboard delivers all outputs in a professional, filter-driven interface — giving revenue teams the analytical clarity they need in minutes, not hours.

## Data Source

**Kaggle — Hotel Booking Demand**
<https://www.kaggle.com/datasets/jessemostipak/hotel-booking-demand>

119,390 real hotel booking records across two properties (City Hotel and Resort Hotel) covering July 2015 – August 2017. 32 fields including ADR, lead time, distribution channel, room type, cancellation status, and guest demographics.

After cleaning (filling 4 null `children`, 488 null `country`, 16 K null `agent` / 112 K null `company`; dropping 2 records with ADR outside 0–5000), **119,388 rows** were loaded.

## Core Business Insights

### Insight 1 — OTA bookings cancel at 2.3× the rate of direct bookings

| Channel | Bookings | Cancellation Rate |
|---------|----------|------------------|
| TA/TO (OTA) | 97,869 | **41.0%** |
| Direct | 14,644 | **17.5%** |

OTA bookings dominate volume (82% of all bookings) but carry a dramatically higher cancellation rate. Every 100 OTA bookings that cancel represent roughly $10,000 in lost committed revenue. Revenue strategy should weight direct-channel acquisition and consider non-refundable deposit incentives for OTA customers with long lead times.

### Insight 2 — Cancellation probability scales monotonically with lead time

| Booking lead time | Bookings | Cancellation Rate |
|------------------|----------|------------------|
| 0–29 days | 38,047 | **18.2%** |
| 30–89 days | 29,918 | **37.8%** |
| 90–179 days | 26,462 | **44.6%** |
| 180+ days | 24,961 | **56.8%** |

A guest booking more than 6 months out is 3× more likely to cancel than a last-minute guest. Overbooking models and deposit-policy tiering should be calibrated against these empirical curves rather than flat assumptions.

### Insight 3 — Negative price elasticity confirmed, but ADR alone explains only 30% of occupancy variance

**β = −0.114 percentage points per $1 ADR · R² = 0.302**

A $10 increase in ADR is associated with a ~1.1 pp drop in occupancy rate. The moderate R² signals that ADR is a real but partial driver — seasonal demand, channel mix, and lead-time composition explain the remaining variance. Pricing decisions should combine the rule-based engine (Modules 3) with seasonal context rather than treating ADR as the sole lever.

## How to Run

**1. Install dependencies**

```bash
pip install pandas scikit-learn
```

SQLite is part of the Python standard library — no separate installation needed.

**2. Run the pipeline**

```bash
cd hotel-revenue-intelligence

# Ingest CSV → SQLite
python3 scripts/load_data.py

# Compute KPIs + write JSON
python3 scripts/analysis.py

# Apply pricing rules + append to JSON
python3 scripts/pricing_engine.py
```

**3. Launch the dashboard**

```bash
# Must be run from project root so ./output/kpi_data.json resolves correctly
python3 -m http.server 8080
```

Open **<http://localhost:8080/dashboard/index.html>** in your browser.
